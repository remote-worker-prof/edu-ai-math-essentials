# Индекс run-артефактов `ЛР02 GPU` (для студентов)

Этот файл помогает быстро понять, какие каталоги запусков есть в репозитории и
какие файлы в них считать основными.

## Что есть в `runs_*`
- `runs_acceptance/` — короткий приёмочный прогон, где проверялся общий маршрут.
- `runs_quick/` — быстрые тестовые прогоны с маленьким бюджетом времени.
- `runs_quick_tf/` — быстрые прогоны с вариантами профиля и диагностикой порогов.
- `runs_no_gpu_expected_fail/` — негативный сценарий без доступного GPU.
- `runs_stability_a/` — стабильный длительный прогон для фиксации учебного эталона.

## Какие файлы главные
- `*.summary.json` — главный итог прогона (PASS/FAIL и метрики).
- `*.executed.ipynb` — технический артефакт с полным выводом ячеек.
- `run*.log` — консольный журнал запуска скрипта.

## Как студенту читать результат
Сначала смотрите `summary.json`:
1. `overall_pass`
2. `success_count` и `prompt_count`
3. `mean_match_ratio`
4. `test_perplexity` и `baseline_perplexity`
5. `stop_reason`

Потом, при необходимости:
- открывайте `*.executed.ipynb` для детального разбора;
- сверяйтесь с `run*.log`, если нужно увидеть сценарий запуска скрипта.

## Важное замечание
Для прохождения `ЛР02 GPU` жёсткий критерий остаётся прежним:
- `success_count >= 19` (из 20).
