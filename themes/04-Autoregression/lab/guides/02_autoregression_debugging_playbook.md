# Диагностический порядок: `04-Autoregression / ЛР01`

## Принцип работы с ошибками
Всегда проверяйте блоки в фиксированном порядке:
1. данные;
2. маска;
3. модель;
4. метрики;
5. генерация;
6. внимание.

Такой порядок снижает число ложных гипотез и ускоряет исправление.

## 1. Данные и сдвиг
Симптомы:
- форма входа и цели не совпадает по длине;
- метрики не растут с первых эпох;
- разные запуски дают разные первые примеры.

Проверки:
- `X.shape == Y.shape`;
- `Y[:, :-1]` совпадает с `X[:, 1:]` на контрольных примерах;
- повторная генерация с тем же `seed` даёт идентичные данные.

## 2. Причинная маска
Симптомы:
- модель обучается слишком «идеально» уже в начале;
- диагностика показывает доступ к будущим токенам.

Проверки:
- маска имеет форму `(T, T)`;
- маска нижнетреугольная;
- верхний треугольник содержит только запрещающие значения.

## 3. Декодерный блок
Симптомы:
- ошибка несовпадения размерностей;
- `NaN` в функции потерь;
- нестабильные кривые обучения.

Проверки:
- совпадают размерности входа и выхода блока;
- остаточные связи сложены по одинаковым формам;
- нормализация вызывается после сложения.

## 4. Метрики обучения
Симптомы:
- точность высокая, а перплексия аномально большая;
- функция потерь не убывает.

Проверки:
- используется корректная перекрёстная энтропия по токенам;
- перплексия считается как `exp(loss)`;
- исключены паддинговые позиции из расчёта, если это предусмотрено реализацией.

## 5. Детерминированная генерация
Симптомы:
- при одинаковом запуске результат каждый раз разный;
- шаблон редко выполняется.

Проверки:
- используется детерминированный выбор `argmax`;
- начальные подсказки фиксированы;
- максимальная длина генерации и условие остановки заданы явно.

## 6. Диагностика внимания
Симптомы:
- карта внимания визуально кажется случайной;
- видно заметную массу в будущем.

Проверки:
- карта усредняется по головам, если голов несколько;
- анализируется только непустой фрагмент;
- доля массы в верхнем треугольнике близка к нулю.
