# Диагностический порядок: `04-Autoregression / ЛР01`

## Принцип
Всегда проверяйте блоки в фиксированной последовательности:
1. данные;
2. маска;
3. модель;
4. метрики;
5. генерация;
6. внимание.

## 1. Данные и сдвиг
Симптомы:
- `X` и `Y` не совпадают по форме;
- запуски с одним `seed` дают разные первые примеры;
- правило перехода фактически зависит от одного токена.

Проверки:
- `X.shape == Y.shape`;
- `Y[:, :-1]` совпадает с `X[:, 1:]`;
- при повторной генерации с тем же `seed` первые `k` примеров идентичны побайтно;
- следующий токен определяется как функция двух предыдущих содержательных токенов.

## 2. Причинная маска
Симптомы:
- метрики становятся «слишком идеальными» слишком рано;
- в диагностике есть внимание в будущее.

Проверки:
- форма маски `(T, T)`;
- маска нижнетреугольная;
- верхний треугольник полностью запрещён.

## 3. Декодерный блок
Симптомы:
- ошибки размерностей;
- нестабильные кривые;
- `NaN` в функции потерь.

Проверки:
- комбинированная маска учитывает и ось запросов, и ось ключей;
- остаточные связи складывают тензоры одной формы;
- нормализация вызывается после сложения.

## 4. Метрики
Симптомы:
- высокая точность при аномально высокой перплексии;
- функция потерь не снижается.

Проверки:
- корректная перекрёстная энтропия по токенам;
- `perplexity = exp(loss)`;
- дополнение `PAD` исключено из метрик.

## 5. Генерация
Симптомы:
- результат меняется между одинаковыми запусками;
- порог `18/20` не достигается.

Проверки:
- выбор следующего токена выполнен через `argmax`;
- подсказки фиксированы;
- сравнение идёт по целому контролируемому продолжению, а не только по первому токену.

## 6. Диагностика внимания
Симптомы:
- карта внимания выглядит случайной;
- заметная масса в будущем.

Проверки:
- веса усреднены по головам;
- анализируется только непустой фрагмент;
- отношение массы в верхнем треугольнике ниже заданного порога.
