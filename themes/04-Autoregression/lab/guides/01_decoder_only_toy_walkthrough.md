# Пошаговый разбор: `04-Autoregression / ЛР01`

## Канонический порядок работы
1. Прочитать `00_autoregression_prerequisites.md`.
2. Открыть учебную тетрадь `01_decoder_only_causal_toy.ipynb`.
3. Выполнять `TODO` строго по порядку от 1 до 6.
4. После каждого `TODO` запускать мини-проверку из следующей ячейки.
5. При затруднениях перейти в `02_autoregression_debugging_playbook.md`.
6. Только после второй попытки смотреть тетрадь-решение (solution notebook).

## Зачем эта лабораторная
Эта работа вводит генеративный режим без лишней сложности:
- короткий синтетический корпус;
- полностью воспроизводимый маршрут;
- прозрачные математические проверки.

Главная методическая цель: закрепить причинную маску как обязательный механизм честного предсказания следующего токена.

## Что делает каждый `TODO`
### TODO 1: данные и сдвиг
- построить детерминированный корпус;
- получить пары `(вход, цель)` со сдвигом на один шаг;
- проверить побайтную воспроизводимость первых `k` примеров.

### TODO 2: причинная маска
- построить нижнетреугольную маску;
- проверить форму и булев тип;
- убедиться, что будущие позиции закрыты.

### TODO 3: декодерный блок
- реализовать маскированное самовнимание;
- добавить остаточные связи и нормализацию;
- вернуть, при необходимости, веса внимания для диагностики.

### TODO 4: модель и обучение
- собрать модель предсказания следующего токена;
- обучить модель на фиксированном наборе;
- построить кривые `loss` и `perplexity`.

### TODO 5: детерминированная генерация
- запустить генерацию по фиксированным подсказкам;
- использовать детерминированный выбор следующего токена;
- посчитать долю корректных последовательностей.

### TODO 6: диагностика внимания
- извлечь карту внимания на одном примере;
- проверить отсутствие массы в верхнем треугольнике;
- сформулировать вывод, что модель не смотрит в будущее.

## Контрольные ориентиры
Ожидаемый итоговый уровень:
- `test_token_accuracy >= 0.97`;
- `test_perplexity <= 1.30`;
- шаблон корректен минимум в `18/20` детерминированных генераций.

## Частые методические ошибки
- студент проверяет только точность, но не проверяет структуру маски;
- генерация запускается в случайном режиме, что ломает воспроизводимость;
- карты внимания читаются без проверки верхнего треугольника.

## Что должно остаться после выполнения
- уверенное понимание связки «сдвиг данных + причинная маска + генерация»;
- умение объяснить, почему низкая перплексия и высокая токенная точность должны идти вместе на этой задаче.
