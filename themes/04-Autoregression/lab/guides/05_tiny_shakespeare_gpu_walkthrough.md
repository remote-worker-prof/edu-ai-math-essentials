# Пошаговый разбор: `04-Autoregression / ЛР02 GPU` на `Tiny Shakespeare`

## Канонический порядок
1. Завершить `ЛР02 CPU` и сохранить итоговую перплексию.
2. Открыть `02_decoder_only_tiny_shakespeare_gpu.ipynb`.
3. Выполнять `TODO` строго от 1 до 6.
4. После каждого шага запускать мини-проверки.
5. При затруднениях использовать `06_tiny_shakespeare_gpu_debugging_playbook.md`.

## Методическая цель
`ЛР02 GPU` не меняет математическую постановку, а проверяет масштабирование той же постановки:
- больше данных;
- больший контекст;
- более ёмкая модель;
- сравнение с CPU-ориентиром качества.

## Требование к среде выполнения
`GPU`-тетрадь не содержит резервного CPU-контура. Если графический процессор недоступен:
- подготовьте среду с корректно установленными CUDA-драйверами;
- убедитесь, что `tf.config.list_physical_devices('GPU')` возвращает непустой список;
- только после этого запускайте обучение и контрольные проверки.

Перед запуском выставьте:
- `COURSE_RUNTIME_MODE=local-gpu`.

Используйте уже подготовленное окружение из `00-Foundations`:
- [../../../00-Foundations/guides/05_local_tensorflow_gpu_notebooks.md](../../../00-Foundations/guides/05_local_tensorflow_gpu_notebooks.md)
- [../../../00-Foundations/guides/06_tensorflow_cuda_version_selection.md](../../../00-Foundations/guides/06_tensorflow_cuda_version_selection.md)
- [../../../00-Foundations/guides/07_tensorflow_blackwell_local_gpu_case_study.md](../../../00-Foundations/guides/07_tensorflow_blackwell_local_gpu_case_study.md)
- [07_lr04_local_gpu_tf221_checklist.md](07_lr04_local_gpu_tf221_checklist.md)

В рамках ЛР не предполагаются системные правки ОС и команды через `pkexec`.

### Recovery для текущей `.venv` (только при провале `gpu_preflight()`)
Сначала повторно используйте уже подготовленное окружение. Если `gpu_preflight()` не проходит, выполните в активной `.venv`:

```bash
python3 -m pip install --upgrade 'tensorflow[and-cuda]>=2.16,<2.20'
python3 -m pip install -r themes/04-Autoregression/lab/requirements.txt
```

Если `gpu_preflight()` уже проходит, повторную переустановку делать не нужно.

Для видеокарт `RTX 50xx`, если после базового recovery остаётся ошибка
совместимости (`CUDA_ERROR_INVALID_PTX`, `CUDA_ERROR_INVALID_HANDLE`), выполните:

```bash
python3 -m pip install --upgrade 'tensorflow[and-cuda]==2.21.0'
python3 -m pip install --upgrade 'nvidia-cuda-nvcc-cu12>=12.8'
```

После обновления перезапустите ядро и повторите `gpu_preflight()`.
GPU-ноутбук автоматически предзагружает CUDA-библиотеки из `.venv` перед импортом TensorFlow.

## Критерии прохождения
- `gpu_preflight()` пройден полностью;
- `test_perplexity < baseline_perplexity`;
- `success_count >= 19` (из `20`) в режиме контролируемого продолжения;
- `mean_match_ratio >= cfg['gen_mean_threshold']` в режиме контролируемого продолжения;
- warm-up выполнен отдельно и не входит в измеряемый бюджет;
- измеряемый бюджет после warm-up равен `60` минутам;
- обучение завершено по ранней остановке, достижению генерационной цели или по лимиту времени;
- диагностика внимания проходит;
- `test_perplexity < CPU_REFERENCE_PERPLEXITY` фиксируется как индикатор качества.
