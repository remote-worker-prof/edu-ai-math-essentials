# Диагностический порядок: `04-Autoregression / ЛР02`

## Принцип
Проверяйте этапы в порядке:
1. корпус и словарь;
2. окна и разбиение;
3. маска и модель;
4. метрики и базовый ориентир;
5. генерация;
6. внимание.

## 1. Корпус и словарь
Симптомы:
- разные запуски дают разный словарь символов;
- появляются неожиданные идентификаторы токенов.

Проверки:
- словарь строится детерминированно (например, через сортировку символов);
- `char_to_id` и `id_to_char` взаимно согласованы;
- размер словаря стабилен между запусками профиля.

## 2. Окна и разбиение
Симптомы:
- `X` и `Y` разных размеров;
- не выполняется сдвиг на один шаг;
- качество нестабильно при одинаковом `seed`.

Проверки:
- `X.shape == Y.shape`;
- `Y[:, :-1] == X[:, 1:]` на контрольных примерах;
- разбиение выполнено индексами без случайного перемешивания.

## 3. Маска и декодер
Симптомы:
- модель «угадывает» слишком рано;
- карта внимания показывает будущее.

Проверки:
- причинная маска нижнетреугольная;
- комбинированная маска учитывает обе оси;
- форма `attention_scores` соответствует ожиданию.

## 4. Метрики
Симптомы:
- перплексия не лучше базового ориентира;
- обучение не сходится.

Проверки:
- правильная функция потерь по токенам;
- корректный расчёт `perplexity = exp(loss)`;
- сравнение с базовым ориентиром рассчитано на том же тестовом срезе.

## 5. Генерация
Симптомы:
- одинаковая подсказка даёт разные результаты;
- шаблон качества не выполняется.

Проверки:
- используется `argmax` без случайной выборки;
- набор подсказок фиксирован;
- длина генерации фиксирована.

## 6. Диагностика внимания
Симптомы:
- верхний треугольник тепловой карты содержит заметную массу.

Проверки:
- усреднение по головам сделано корректно;
- анализ ограничен непустым фрагментом;
- отношение массы в будущем ниже заданного порога.

## Чек-лист перед завершением
1. Пройден профиль `CPU-friendly`.
2. Для `GPU-friendly` показано улучшение относительно `CPU-friendly`.
3. Выполнены пороги генерации.
4. Диагностика внимания подтверждает отсутствие доступа к будущим позициям.
