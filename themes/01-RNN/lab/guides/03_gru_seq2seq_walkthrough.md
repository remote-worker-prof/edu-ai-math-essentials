## Что меняется после `01-RNN / ЛР02`
Главный переход такой:
- раньше модель выдавала ответ для того же временного ряда;
- теперь модель получает одну последовательность и генерирует другую.

Из-за этого появляются новые сущности:
- `encoder_input`;
- `decoder_input`;
- `decoder_target`;
- словарь и специальные токены `PAD`, `SOS`, `EOS`;
- строгая метрика `exact_match`.

## TODO 1-3: reverse-последовательность и сдвиг decoder
### Что нужно понять
- `encoder_input` хранит входную последовательность;
- `decoder_input` начинается с `SOS`, а затем получает развёрнутую последовательность;
- `decoder_target` совпадает с тем, что decoder должен предсказать: `rev + [EOS]`.

### Логика решения
- `rev` строится как обратный порядок токенов;
- `dec_in = [SOS] + rev`;
- `dec_out = rev + [EOS]`;
- итоговые формы должны быть `(N, T_in)`, `(N, T_out)`, `(N, T_out, 1)`.

### Ожидаемые результаты
- `enc_train.shape == (N_train, ENC_LEN)`
- `dec_in_train.shape == (N_train, DEC_LEN)`
- `dec_tgt_train.shape == (N_train, DEC_LEN, 1)`

### Типичные ошибки
- забыть развернуть последовательность;
- перепутать `decoder_input` и `decoder_target`;
- забыть последнюю ось у `decoder_target`.

### Self-check
- Почему `decoder_input` не может быть равен `decoder_target`?
  Потому что decoder на шаге `t` должен получать предыдущий правильный токен и предсказывать следующий.
- Почему `decoder_target` хранится как `(N, T_out, 1)`?
  Потому что `sparse_categorical_crossentropy` ждёт целочисленный индекс класса на каждом шаге.

## TODO 4-9: сборка encoder-decoder модели
### Что нужно понять
- encoder должен превратить токены во векторы через `Embedding`;
- `GRU` encoder читает вход и отдаёт финальный контекст;
- decoder получает свой `Embedding`, затем `GRU` с `initial_state=context`;
- на каждом шаге нужна вероятность по всему словарю, значит финальный слой — `Dense(VOCAB_SIZE, activation='softmax')`.

### Логика решения
- собрать `encoder_seq` с `Embedding` и `GRU`;
- собрать `decoder_seq` с `Embedding`;
- создать `dec_gru` с `return_sequences=True`;
- построить `head_seq` с `Dense(VOCAB_SIZE, activation='softmax')`;
- скомпилировать модель как задачу классификации по словарю.

### Ожидаемые результаты
- `model.output_shape == (None, DEC_LEN, VOCAB_SIZE)`
- мини-проверка модели проходит;
- понятно, почему в выходе появляется ось словаря `V`.

### Типичные ошибки
- забыть `return_sequences=True` у decoder;
- вернуть один вектор вместо последовательности распределений;
- использовать `binary_crossentropy` вместо `sparse_categorical_crossentropy`.

### Self-check
- Почему здесь нельзя закончить модель одним нейроном `Dense(1)`?
  Потому что на каждом шаге нужно выбрать один токен из словаря, а не бинарный класс.

## TODO 10: обучение
### Что нужно понять
- в `fit` подаются два входа: `[encoder_input, decoder_input]`;
- целевой тензор — `decoder_target`;
- `validation_split=0.2` используется так же, как раньше, но теперь метрика считается по токенам.

### Ожидаемые результаты
- в `history.history` есть `loss`, `val_loss`, `accuracy`, `val_accuracy`;
- `token_accuracy` постепенно растёт;
- история не даёт `exact_match`: его считаем отдельно вручную.

### Типичные ошибки
- передать в `fit` только один вход;
- перепутать порядок `[dec_in_train, enc_train]`;
- не использовать аргумент `epochs`.

## TODO 11: `exact_match`
### Что нужно понять
- `argmax` переводит распределение по словарю в индекс токена;
- `exact_match` должен сравнивать всю последовательность целиком;
- PAD-позиции нельзя учитывать как полезные токены.

### Логика решения
- взять `preds = probs.argmax(axis=-1)`;
- вытащить `target = dec_tgt_test[:, :, 0]`;
- построить `mask = (target != PAD_ID)`;
- для каждого объекта проверить, совпали ли все значимые позиции.

### Worked example
Пусть:

```text
target = [7, 5, 2, EOS, PAD]
preds  = [7, 5, 2, EOS,  3 ]
mask   = [1, 1, 1,   1,  0 ]
```

Тогда:
- по значимым позициям всё совпало;
- последняя PAD-позиция игнорируется;
- вклад в `exact_match` равен `1`.

### Типичные ошибки
- сравнивать `preds` и `target` без маски;
- усреднять по токенам вместо усреднения по объектам;
- забыть убрать последнюю ось у `decoder_target`.

### Self-check
- Почему `token_accuracy` может быть высокой, а `exact_match` низкой?
  Потому что последовательность считается правильной только если совпали все значимые токены, а не большинство из них.

- почему выход имеет форму `(N, T_out, V)`;
