## Интуиция на пальцах
Весь курс можно читать как один повторяющийся цикл:
1. у нас есть последовательность токенов или признаков;
2. мы явно задаём контракт данных (что подаём на вход и что считаем целью);
3. модель строит вероятности/предсказания;
4. мы проверяем метрики и диагностику, чтобы убедиться, что качество выросло честно.

Главный принцип: теория не живёт отдельно от кода. Каждый новый термин сразу привязываем к конкретному `TODO` и мини-проверке в лабораторной тетради.

## Контракт данных (оси, маски, inputs/targets)
Базовые формы, которые повторяются в `00-05`:

- признаки во времени: `(batch, time, features)`
- токены: `(batch, time)`
- token-level logits: `(batch, time, vocab_size)`
- many-to-one logits: `(batch, num_classes)` или `(batch, 1)`

Базовые маски:
- `padding_mask`: какие позиции не `PAD`, обычно `(batch, time)`
- `causal_mask`: запрет смотреть в будущее, обычно `(time, time)` нижний треугольник
- `cross_attention_mask`: какие позиции `encoder_input` доступны decoder, обычно `(batch, target_time, source_time)`

## Формализация (минимум формул)
### Математический минимум sequence/attention/transformer

1. Рекуррентный шаг:
   `h_t = f(x_t, h_{t-1})`
2. Attention:
   `scores = softmax(QK^T / sqrt(d_k))`
3. Context:
   `context = scores @ V`
4. Transformer-блок:
   `h1 = LayerNorm(x + Attention(x))`
   `h2 = LayerNorm(h1 + FFN(h1))`

Этого минимума достаточно, чтобы осмысленно читать все лабораторные темы курса.

## Ручной мини-пример
Пусть у нас mini-batch из двух строк токенов:

```text
encoder_input =
[
  [12,  7,  5,  0,  0],
  [ 3, 18, 11,  4,  2],
]

padding_mask =
[
  [1, 1, 1, 0, 0],
  [1, 1, 1, 1, 1],
]
```

Для decoder:

```text
decoder_input  = [SOS, y1, y2, y3]
decoder_target = [y1,  y2, y3, y4]
```

Идея shift-контракта: в `decoder_input` мы даём предыдущий правильный токен, а в `decoder_target` просим предсказать следующий.

## Внешний обзор архитектур (вне основного трека)
Ниже обзор, чтобы студент видел всю карту DL-подходов, но не путал её с текущим лабораторным маршрутом.

- `MLP`: универсальный базовый блок для векторных признаков.
- `CNN`: локальные фильтры и иерархия признаков в изображениях/сигналах.
- `GNN`: распространение информации по графу (узлы/рёбра).
- `Autoencoder`: сжатие и восстановление, обучение представлений.
- `GAN`: генератор и дискриминатор в состязательном цикле.
- `Diffusion`: поэтапная генерация из шума через обратный процесс.

Эти направления важны для кругозора, но в текущем курсе `EAIME` лабораторная нить остаётся сфокусированной на sequence/attention/transformer-маршруте.
