# Beginner Guide: Self-Attention и Positional Embedding без перегруза

## Главная идея
В attention-теме decoder спрашивал:

```text
На какую часть входа мне сейчас смотреть?
```

В Transformer encoder каждый токен делает почти то же самое, но уже внутри одной последовательности:

```text
На какие другие токены мне сейчас стоит смотреть?
```

Это и есть `self-attention`.

## Как это представить без формул
Пусть есть короткая последовательность:

```text
[7, 5, 2, 3]
```

Если модель анализирует токен `7`, ей может быть полезно посмотреть:
- где находится `3`;
- какие токены стоят между ними;
- есть ли важный шаблон по позициям.

Self-attention позволяет токену не жить изолированно, а собирать информацию из остальных позиций.

## Почему порядок нельзя потерять
Если взять две последовательности:

```text
[7, 5, 2, 3]
[3, 5, 2, 7]
```

Набор токенов один и тот же, но смысл задачи может быть разным.

Если модель знает только “какие токены были”, но не знает “где они были”, она не отличит эти случаи.

Поэтому к token embeddings добавляют position embeddings.

## `TokenAndPositionEmbedding`
Самая удобная учебная схема:
- `token_embedding` превращает token id в вектор;
- `position_embedding` превращает номер позиции в вектор;
- затем оба вектора складываются.

Итог:

```text
embedding[token] + embedding[position]
```

Так модель видит и содержимое токена, и его место в последовательности.

## Что делает `TransformerEncoderBlock`
Минимальный encoder block состоит из:
1. `MultiHeadAttention`
2. residual connection
3. `LayerNormalization`
4. feed-forward network
5. ещё одной residual + normalization

Полезная интуиция:
- attention смешивает информацию между позициями;
- feed-forward дообрабатывает каждый шаг;
- residual помогает не потерять старую информацию;
- normalization делает обучение стабильнее.

## Где здесь `multi-head`
Одна голова внимания — один способ посмотреть на последовательность.

Несколько голов полезны потому, что:
- одна может замечать локальные зависимости;
- другая — дальние;
- третья — служебные токены или границы.

В наших лабораторных heads немного, потому что задача учебная, а не индустриальная.

## Padding mask простыми словами
Если последовательность выглядит так:

```text
[7, 5, 2, 3, 0, 0, 0]
```

то последние нули — не настоящие данные.

Значит:
- attention не должен на них смотреть;
- pooled-представление не должно по ним усредняться.

Именно для этого нужна `padding mask`.

## Как читать `attention_scores`
В Keras `MultiHeadAttention` trace часто возвращает:

```text
(batch, heads, T, T)
```

Это значит:
- для каждого объекта в batch;
- для каждой attention-head;
- для каждой query-позиции;
- есть веса по всем key-позициям.

Для visual interpretation часто удобно:
- выбрать один объект;
- усреднить по головам;
- получить тепловую карту `(T, T)`.

## Что должно остаться в голове
- self-attention = токены смотрят друг на друга внутри одной последовательности;
- positional embedding = способ вернуть модели чувство порядка;
- Transformer encoder block = attention + feed-forward + residual + normalization;
- без mask padded позиции портят и обучение, и интерпретацию.
