# Диагностический порядок: `04-Autoregression / ЛР02 GPU`

## Принцип
Порядок проверки:
1. режим запуска и конфигурация;
2. корпус и окна;
3. декодер и маски;
4. метрики и сравнение с CPU;
5. генерация;
6. внимание.

## 1. Режим запуска
Симптомы:
- тетрадь завершается ошибкой до обучения;
- TensorFlow не видит ни одного `GPU`.

Проверки:
- проверить, что `COURSE_RUNTIME_MODE=local-gpu`;
- вывести число доступных GPU через `tf.config.list_physical_devices('GPU')`;
- проверить установку CUDA-драйверов и совместимость версии TensorFlow;
- перезапустить ядро после исправления среды.

### 1.1. Сценарий: IDE/окно упало, но расчёт мог завершиться
Симптомы:
- окно VS Code аварийно закрылось (`code: 139`);
- `run*.log` выглядит слишком коротким;
- есть подозрение, что `nbconvert` дошёл до конца, но отчёт не дописался.

Порядок восстановления:
1. Проверить, что выполненная тетрадь `*.executed.ipynb` существует в каталоге запуска.
2. Извлечь итог из тетради отдельной утилитой:
```bash
python3 themes/04-Autoregression/lab/scripts/extract_gpu_run_summary.py \
  <path-to-executed.ipynb> \
  --output <path-to-summary.json>
```
3. Читать итоговый статус только по `summary.json`.

Почему это корректно:
- `GPU_RUN_SUMMARY_JSON` печатается внутри самой тетради;
- значит источник истины — выполненная тетрадь, а не полнота консольного лога.

Опорные материалы для диагностики:
- [../../../00-Foundations/guides/05_local_tensorflow_gpu_notebooks.md](../../../00-Foundations/guides/05_local_tensorflow_gpu_notebooks.md)
- [../../../00-Foundations/guides/06_tensorflow_cuda_version_selection.md](../../../00-Foundations/guides/06_tensorflow_cuda_version_selection.md)
- [../../../00-Foundations/guides/07_tensorflow_blackwell_local_gpu_case_study.md](../../../00-Foundations/guides/07_tensorflow_blackwell_local_gpu_case_study.md)

Recovery для текущей `.venv` (только при провале `gpu_preflight()`, без `pkexec` и правок ОС):

```bash
python3 -m pip install --upgrade 'tensorflow[and-cuda]>=2.16,<2.20'
python3 -m pip install -r themes/04-Autoregression/lab/requirements.txt
```

Если `gpu_preflight()` уже проходит, повторная переустановка не требуется.

Если проблема остаётся на `RTX 50xx` и видны признаки compatibility-level
ошибки (`INVALID_PTX`, `INVALID_HANDLE`, сбой XLA-autotuner), примените:

```bash
python3 -m pip install --upgrade 'tensorflow[and-cuda]==2.21.0'
python3 -m pip install --upgrade 'nvidia-cuda-nvcc-cu12>=12.8'
```

После обновления обязательно перезапустите ядро и повторите `gpu_preflight()`.
В GPU-ноутбуках встроена предзагрузка CUDA-библиотек из `.venv` перед импортом TensorFlow.

## 2. Корпус и окна
Симптомы:
- размер данных не соответствует конфигурации;
- окна построены с ошибкой сдвига.

Проверки:
- длина среза текста совпадает с `cfg['chars']`;
- `X.shape == Y.shape`;
- сдвиг `Y[:, :-1] == X[:, 1:]` соблюдается.

## 3. Декодер и маски
Симптомы:
- нестабильная сходимость;
- заметная масса внимания в будущем.

Проверки:
- причинная маска нижнетреугольная;
- маска учитывает обе оси;
- `PAD` исключён из функции потерь и метрик.

## 4. Метрики и сравнение с CPU
Симптомы:
- перплексия хуже частотного ориентира;
- в режиме `gpu` нет улучшения относительно CPU.
- обучение не останавливается в ожидаемом учебном бюджете.

Проверки:
- warm-up действительно выполняется до запуска измеряемого бюджета;
- валидация вызывается каждые `cfg['eval_every_steps']` шагов;
- generation-probe строится по `starts_val`, а не по `starts_test`;
- ранняя остановка срабатывает по `val_loss` как вспомогательное условие, не ломая цель `19/20`;
- остановка по generation-gate срабатывает только после минимального времени timed-run
  и нескольких подряд успешных probe-проверок;
- лимит времени (`cfg['max_training_minutes']`) ограничивает только измеряемую фазу после warm-up;
- `test_perplexity < baseline_perplexity`;
- статус `test_perplexity < CPU_REFERENCE_PERPLEXITY` фиксируется как индикатор, а не как аварийный барьер.

## 5. Генерация
Симптомы:
- нестабильная генерация между одинаковыми запусками;
- не достигаются пороги качества.
- цель `19/20` не достигается даже при длительном запуске.

Проверки:
- проходной критерий считается в режиме контролируемого продолжения (teacher forcing);
- свободная генерация через `argmax` используется как демонстрационный блок;
- подсказки фиксированы;
- достигается `success_count >= 19` (из `20`);
- проходит порог `mean_match_ratio`.

## 6. Диагностика внимания
Симптомы:
- верхний треугольник содержит заметную массу.

Проверки:
- вычислена усреднённая карта внимания;
- отношение массы в будущем ниже порога;
- вывод проверки сохранён в отчёте.

## Финальный чек-лист
1. `gpu_preflight()` пройден полностью.
2. Пройдены метрики качества и генерации для этого режима.
3. Подтверждена корректность причинной маски и диагностики внимания.
4. Подтверждено улучшение относительно CPU-ориентира.
5. Если лог запуска неполный, итог восстановлен из `*.executed.ipynb` в `summary.json`.
