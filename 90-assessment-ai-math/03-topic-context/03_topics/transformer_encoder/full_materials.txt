## Интуиция задачи без формул

Здесь модель решает очень специальную задачу:
- во входной последовательности точно есть токены `7` и `3`;
- метка зависит не от того, что эти токены вообще были,
- а от того, **кто встретился раньше**.

Пример:

```text
[7, 5, 2, 3, PAD, PAD, ...] -> y = 1
[3, 5, 2, 7, PAD, PAD, ...] -> y = 0
```

Набор токенов одинаковый, но порядок разный.
Это и есть главная причина, по которой Transformer encoder не может обходиться без positional embedding.

## Что изменилось после `02-Attention / ЛР01`

В attention-лаборатории decoder смотрел на encoder.

Теперь:
- рекуррентной ячейки больше нет;
- одна и та же последовательность сама строит `query`, `key`, `value`;
- вместо `cross-attention` мы используем `self-attention`;
- positional embedding становится обязательной частью модели.

[cell 008 | markdown]
## Контракт данных

Вход:
- padded последовательности целых token ids,
- форма `X -> (N, T)`.

Выход:
- бинарная метка `y -> (N,)`.

Правило метки:
- `y = 1`, если `7` стоит раньше `3`;
- `y = 0`, если `3` стоит раньше `7`.

[cell 009 | markdown]
## Таблица форм тензоров

| Сущность | Смысл | Форма |
|---|---|---|
| `X_train` | padded token ids | `(N, T)` |
| `padding_mask` | полезные позиции | `(N, T)` |
| `embeddings` | token + position embeddings | `(N, T, E)` |
| `attention_scores` | веса внимания по головам | `(N, H, T, T)` |
| `pooled` | один вектор на объект | `(N, E)` |
| `y_pred` | вероятность класса | `(N, 1)` |

[cell 010 | markdown]
## Шпаргалка по обозначениям и формам

- `N` — число объектов.
- `T` — длина последовательности после padding.
- `E` — размер embedding / model dimension.
- `H` — число attention heads.
- `PAD = 0`.

Практический фокус этой ЛР:
- проверить shapes;
- проверить mask;
- проверить, что attention не уходит в padded хвост.

[cell 011 | markdown]
## Контракт модели

Модель должна состоять из таких блоков:
1. `TokenAndPositionEmbedding`
2. `TransformerEncoderBlock`
3. masked average pooling
4. classifier head для binary classification

Отдельно нужен tracing-path, который возвращает `attention_scores` хотя бы для одного encoder block.

[cell 012 | markdown]
## Мини-теория

Минимальный encoder block:

$$
H_1 = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))
$$

$$
H_2 = \mathrm{LayerNorm}(H_1 + \mathrm{FFN}(H_1))
$$

Если positional embedding отсутствует, то attention знает “какие токены были”, но слабо знает “где они были”.
Для этой лабораторной это критично, потому что метка зависит именно от порядка.

def masked_average(x, mask):
    """Вычисляет среднее по времени с учётом маски.

    Аргументы:
      x: Тензор признаков формы `(batch, time, embed_dim)`.
      mask: Булева маска полезных позиций формы `(batch, time)`.

    Возвращает:
      Усреднённый тензор формы `(batch, embed_dim)`.

    Исключения:
      ValueError: Если входной тензор имеет неверный ранг.
    """
    if x.shape.rank != 3:
        raise ValueError('Ожидается ранг 3 для тензора признаков.')
    mask = tf.cast(mask, x.dtype)
    mask = tf.expand_dims(mask, axis=-1)
    summed = tf.reduce_sum(x * mask, axis=1)
    counts = tf.reduce_sum(mask, axis=1)
    return summed / tf.maximum(counts, 1.0)


class TransformerEncoderBlock(layers.Layer):
    """Минимальный блок кодировщика трансформера.

    Аргументы:
      embed_dim: Размерность входных признаков.
      num_heads: Число голов внимания.
      ff_dim: Размер скрытого слоя позиционно-независимой сети.
      rate: Доля выключаемых нейронов в прореживании.
      **kwargs: Дополнительные аргументы базового слоя Keras.

    Возвращает:
      Экземпляр блока кодировщика.

    Исключения:
      ValueError: Если `embed_dim` не делится на `num_heads`.
    """

    def __init__(self, embed_dim, num_heads, ff_dim, rate=0.1, **kwargs):
        super().__init__(**kwargs)
        if embed_dim % num_heads != 0:
            raise ValueError('embed_dim должен делиться на num_heads без остатка.')
        self.att = layers.MultiHeadAttention(
            num_heads=num_heads,
            key_dim=embed_dim // num_heads,
            dropout=rate,
        )
        self.ffn = keras.Sequential(
            [
                layers.Dense(ff_dim, activation='relu'),
                layers.Dense(embed_dim),
            ]
        )
        self.layernorm1 = layers.LayerNormalization(epsilon=1e-6)
        self.layernorm2 = layers.LayerNormalization(epsilon=1e-6)
        self.dropout1 = layers.Dropout(rate)
        self.dropout2 = layers.Dropout(rate)
        self.supports_masking = True

Вход:

- форма `X -> (N, T)`.

Выход:

| Сущность | Смысл | Форма |
|---|---|---|
| `X_train` | padded review tokens | `(N, T)` |
| `padding_mask` | полезные позиции | `(N, T)` |
| `embeddings` | token + position embeddings | `(N, T, E)` |
| `attention_scores` | веса внимания | `(N, H, T, T)` |
| `pooled` | один вектор на review | `(N, E)` |
| `y_pred` | вероятность positive class | `(N, 1)` |

- `N` — число review.
- `T` — длина review после padding.
- `E` — размер embedding / model dimension.
- `H` — число attention heads.
- `V` — размер словаря.

Для этой лабораторной важны две привычки:
- проверять mask;
- не пытаться интерпретировать attention на всём review целиком, если оно слишком длинное.

## Мини-теория

Encoder-only Transformer для классификации можно читать так:
- токены получают embeddings и позиции;
- encoder block смешивает информацию между позициями;
- затем всё сворачивается в один вектор на объект;
- classifier head выдаёт sentiment-вероятность.

На практике это уже рабочий пример того, как self-attention заменяет рекуррентный проход в `many-to-one` задаче.

class TokenAndPositionEmbedding(layers.Layer):
    """Складывает токенные и позиционные векторы."""

    def __init__(self, maxlen, vocab_size, embed_dim, **kwargs):
        """Инициализирует слой встраивания.

        Аргументы:
          maxlen: Максимальная длина последовательности.
          vocab_size: Размер словаря токенов.
          embed_dim: Размерность векторного представления.
          **kwargs: Дополнительные аргументы базового слоя Keras.

        Возвращает:
          Экземпляр слоя.

        Исключения:
          ValueError: Если `embed_dim` меньше 1.
        """
        super().__init__(**kwargs)
        if embed_dim < 1:
            raise ValueError('embed_dim должен быть положительным.')
        self.maxlen = maxlen
        self.vocab_size = vocab_size
        self.embed_dim = embed_dim
        self.token_emb = layers.Embedding(vocab_size, embed_dim, mask_zero=True)
        self.pos_emb = layers.Embedding(maxlen, embed_dim)
        self.supports_masking = True

    def call(self, inputs):
        """Возвращает сумму токенных и позиционных векторов.

        Аргументы:
          inputs: Целочисленные токены формы `(batch, time)`.

        Возвращает:
          Тензор формы `(batch, time, embed_dim)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        positions = tf.range(start=0, limit=tf.shape(inputs)[-1], delta=1)
        position_embeddings = self.pos_emb(positions)
        token_embeddings = self.token_emb(inputs)
        return token_embeddings + position_embeddings

    def compute_mask(self, inputs, mask=None):
        """Переадресует маску непустых токенов.

        Аргументы:
          inputs: Целочисленные токены формы `(batch, time)`.
          mask: Входная маска базового уровня.

        Возвращает:
          Булева маска формы `(batch, time)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        return self.token_emb.compute_mask(inputs)


def masked_average(x, mask):
    """Вычисляет среднее по времени с учётом маски.

    Аргументы:
      x: Тензор признаков формы `(batch, time, embed_dim)`.
      mask: Булева маска полезных позиций формы `(batch, time)`.

    Возвращает:
      Усреднённый тензор формы `(batch, embed_dim)`.

    Исключения:
      ValueError: Если входной тензор имеет неверный ранг.
    """
    if x.shape.rank != 3:
        raise ValueError('Ожидается ранг 3 для тензора признаков.')
    mask = tf.cast(mask, x.dtype)
    mask = tf.expand_dims(mask, axis=-1)
    summed = tf.reduce_sum(x * mask, axis=1)
    counts = tf.reduce_sum(mask, axis=1)
    return summed / tf.maximum(counts, 1.0)


class TransformerEncoderBlock(layers.Layer):
    """Минимальный блок кодировщика трансформера."""

    def __init__(self, embed_dim, num_heads, ff_dim, rate=0.1, **kwargs):
        """Инициализирует слои внимания и позиционно-независимой сети.

        Аргументы:
          embed_dim: Размерность входных признаков.
          num_heads: Число голов внимания.
          ff_dim: Размер скрытого слоя позиционно-независимой сети.
          rate: Доля выключаемых нейронов в прореживании.
          **kwargs: Дополнительные аргументы базового слоя Keras.

        Возвращает:
          Экземпляр блока кодировщика.

        Исключения:
          ValueError: Если `embed_dim` не делится на `num_heads`.
        """
        super().__init__(**kwargs)
        if embed_dim % num_heads != 0:
            raise ValueError('embed_dim должен делиться на num_heads без остатка.')
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.ff_dim = ff_dim
        self.rate = rate
        self.att = layers.MultiHeadAttention(
            num_heads=num_heads,
            key_dim=embed_dim // num_heads,
            dropout=rate,
        )
        self.ffn = keras.Sequential(
            [
                layers.Dense(ff_dim, activation='relu'),
                layers.Dense(embed_dim),
            ]
        )
        self.layernorm1 = layers.LayerNormalization(epsilon=1e-6)
        self.layernorm2 = layers.LayerNormalization(epsilon=1e-6)
        self.dropout1 = layers.Dropout(rate)
        self.dropout2 = layers.Dropout(rate)
        self.supports_masking = True

    def call(self, inputs, mask=None, training=None, return_attention_scores=False):
        """Выполняет прямой проход блока кодировщика.

        Аргументы:
          inputs: Входной тензор формы `(batch, time, embed_dim)`.
          mask: Булева маска формы `(batch, time)`.
          training: Признак режима обучения.
          return_attention_scores: Вернуть ли дополнительно веса внимания.

        Возвращает:
          Либо выходной тензор формы `(batch, time, embed_dim)`,
          либо кортеж `(output, attention_scores)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        attention_mask = None
        if mask is not None:
            key_mask = tf.cast(mask[:, tf.newaxis, :], dtype=tf.bool)
            query_mask = tf.cast(mask[:, :, tf.newaxis], dtype=tf.bool)
            attention_mask = query_mask & key_mask

        if return_attention_scores:
            attention_output, attention_scores = self.att(
                query=inputs,
                value=inputs,
                key=inputs,
                attention_mask=attention_mask,
                return_attention_scores=True,
                training=training,
            )
        else:
            attention_output = self.att(
                query=inputs,
                value=inputs,
                key=inputs,
                attention_mask=attention_mask,
                training=training,
            )

        attention_output = self.dropout1(attention_output, training=training)
        proj_input = self.layernorm1(inputs + attention_output)
        proj_output = self.ffn(proj_input)
        proj_output = self.dropout2(proj_output, training=training)
        output = self.layernorm2(proj_input + proj_output)

        if return_attention_scores:
            return output, attention_scores
        return output

    def compute_mask(self, inputs, mask=None):
        """Пробрасывает временную маску на следующий слой.

        Аргументы:
          inputs: Входной тензор признаков.
          mask: Входная булева маска.

        Возвращает:
          Та же маска без изменений.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        return mask

# Поясняющий разбор: самовнимание и позиционные векторы

## Главная идея
В предыдущем блоке декодер спрашивал: «На какую часть входа смотреть сейчас?»

В блоке трансформера каждый токен задаёт похожий вопрос внутри своей же последовательности: «На какие другие позиции мне опереться?»

Это и есть самовнимание (self-attention).

## Интуиция на коротком примере
Возьмём последовательность:

```text
[7, 5, 2, 3]
```

Для корректного решения задачи токену `7` может быть важно:
- где стоит `3`;
- какие элементы находятся между ними;
- какова локальная структура фрагмента.

Самовнимание позволяет собрать такую информацию напрямую, без пошагового рекуррентного прохода.

## Почему порядок нельзя терять
Сравним:

```text
[7, 5, 2, 3]
[3, 5, 2, 7]
```

Набор токенов один, но порядок различается, а вместе с ним может меняться и ответ.

Поэтому к токенному вектору добавляется позиционный вектор:

```text
embedding[token] + embedding[position]
```

## Что делает слой `TokenAndPositionEmbedding`
1. Преобразует `token_id` в токенный вектор.
2. Преобразует индекс позиции в позиционный вектор.
3. Складывает их поэлементно.

В результате модель одновременно видит «что» и «где».

## Что делает `TransformerEncoderBlock`
Минимальный блок:
1. многоголовое внимание (multi-head attention);
2. остаточная связь;
3. нормализация слоя (layer normalization);
4. двухслойная позиционно-независимая сеть;
5. вторая остаточная связь и нормализация.

## Зачем нужна маска дополнения
Если последовательность дополняется `PAD`, такие позиции нельзя считать содержательными.

Поэтому:
- во внимании `PAD` исключаются маской;
- при итоговом усреднении признаков `PAD` также игнорируются.

Бытовая аналогия: если в анкете пустые поля приравнять к реальным ответам, итоговая статистика будет искажена.

## Как читать `attention_scores`
Типичная форма в Keras:

```text
(batch, heads, T, T)
```

Практически удобно:
1. взять один объект из пакета;
2. усреднить по головам;
3. получить тепловую карту `(T, T)`;
4. анализировать только непустой фрагмент.

## Что должно остаться после чтения
- самовнимание связывает позиции внутри одной последовательности;
- позиционные векторы обязательны для задач, где важен порядок;
- маска дополнения нужна в двух местах: внимание и итоговое усреднение;
- интерпретация карт внимания начинается с проверки форм.

- набор токенов может быть одинаковым;
- порядок токенов разный;
- ответ зависит именно от порядка.

Это делает необходимость позиционного кодирования наглядной, а не декларативной.

## Формализация задачи
Правило метки:
- `y = 1`, если токен `7` встречается раньше токена `3`;
- `y = 0`, если токен `3` встречается раньше токена `7`.

Обязательное условие: оба токена присутствуют в каждой полезной последовательности.

## Что проверить на каждом шаге
### 1. Генератор данных
- длина полезной части меняется в заданном диапазоне;
- `PAD = 0`;
- токены `7` и `3` реально вставлены;
- метка согласована с фактическими индексами этих токенов.

### 2. Слой `TokenAndPositionEmbedding`
- вход имеет форму `(batch, time)`;
- выход имеет форму `(batch, time, embed_dim)`;
- к токенному вектору добавляется позиционный, а не заменяет его.

### 3. Блок `TransformerEncoderBlock`
Контракт:
- многоголовое внимание;
- остаточная связь и нормализация;
- позиционно-независимая сеть;
- вторая остаточная связь и нормализация;
- корректная передача маски.

### 4. Классификационная часть
- маскированное усреднение по полезным позициям;
- плотный слой;
- финальный `sigmoid` для бинарной классификации.

### 5. Диагностика внимания
- отдельный путь трассировки с `return_attention_scores=True`;
- усреднение по головам;
- визуализация только по непустому фрагменту.

## Частые ошибки
- метка считается не по позиции, а по значению;
- маска теряется между слоями;
- внимание строится по хвосту `PAD`;
- усреднение делается по всей длине, включая пустые позиции.

## Как читать хороший результат
- `test_acc >= 0.95`;
- ручные примеры с перестановкой `7` и `3` дают разные ответы;
- карта внимания интерпретируема и не концентрируется на хвосте `PAD`.

## Что должно остаться после выполнения
- понимание, что позиция — обязательная часть представления;
- привычка проверять формы и маску до метрик;
- навык читать карту внимания как проверку механики модели.

### 4. Диагностика внимания
- получить `attention_scores` через трассировочный путь;
- усреднить по головам;
- визуализировать только содержательную часть отзыва.

## Частые ошибки
- перепутана функция потерь и финальный слой;
- маска не участвует в пуллинге;
- визуализация включает хвост `PAD`;
- пытаются «улучшать всё сразу» вместо проверки по этапам.

## Как читать итог
Для учебного режима достаточно, чтобы одновременно выполнялось:
- `test_acc >= 0.75`;
- модель заметно лучше случайного угадывания;
- карта внимания не уходит в дополненные позиции;
- студент может объяснить, как переиспользуется блок из `ЛР01`.

## Что должно остаться после выполнения
- понимание переноса от синтетики к реальному тексту;
- привычка проверять не только метрику, но и механику внимания;
- готовность к следующему шагу: декодерный режим и причинная маска.

# Диагностический порядок для `03-Transformer`

## Зачем нужен этот файл
В лабораторных по трансформеру типовые ошибки чаще всего лежат не в общей идее, а в трёх точках:
- формы тензоров;
- позиционные векторы;
- маска дополнения.

Поэтому нужен фиксированный порядок диагностики.

## Базовый режим работы
1. Запускать тетрадь сверху вниз.
2. После правок в середине выполнять `Restart & Run All`.
3. Сначала проверять формы и маску, затем анализировать метрики.

## Контрольный набор печати
```python
print(X_batch.shape)
print(mask.shape)
print(embedded_batch.shape)
print(attention_scores.shape)
```

Ожидаемые формы для кодировщика:
- `tokens -> (batch, T)`
- `mask -> (batch, T)`
- `embeddings -> (batch, T, E)`
- `attention_scores -> (batch, heads, T, T)`

## Порядок локализации ошибок
### 1. Данные
Симптомы:
- метрики около случайного уровня;
- ручной пример противоречит метке;
- `PAD` участвует как обычный токен.

Проверки:
- вывести несколько примеров;
- вручную проверить правило метки;
- убедиться в корректном дополнении до длины `T`.

### 2. Позиционные векторы
Симптомы:
- модель плохо различает перестановки;
- форма после слоя встраивания не `(batch, T, E)`.

Проверки:
- позиции строятся по фактической длине;
- размерности токенных и позиционных векторов совпадают;
- используется сложение, а не конкатенация.

### 3. Маска внимания
Симптомы:
- внимание уходит в хвост `PAD`;
- итоговое усреднение меняется из-за длины дополнения.

Проверки:
- базовая маска: `tokens != 0`;
- маска передаётся в блок внимания;
- маска участвует в пуллинге.

### 4. Классификационная часть
Симптомы:
- несогласованность форм на `fit`;
- функция потерь не соответствует выходу.

Проверки:
- бинарный выход и `binary_crossentropy` согласованы;
- форма `y` корректна;
- размер итогового вектора перед классификатором ожидаем.

## Минимальный маршрут восстановления
1. `Restart & Run All`.
2. Проверить формы тензоров.
3. Проверить маску.
4. Проверить один ручной пример.
5. Проверить карту внимания.
6. Сверяться с решением только после двух попыток.

## Что должно остаться
- привычка диагностировать по этапам, а не по ощущениям;
- понимание, что большинство ошибок в трансформере связано с формами и масками;
- устойчивый маршрут: данные → встраивание → маска → внимание → пуллинг → метрики.

## Интуиция задачи без формул

Здесь модель решает очень специальную задачу:
- во входной последовательности точно есть токены `7` и `3`;
- метка зависит не от того, что эти токены вообще были,
- а от того, **кто встретился раньше**.

Пример:

```text
[7, 5, 2, 3, PAD, PAD, ...] -> y = 1
[3, 5, 2, 7, PAD, PAD, ...] -> y = 0
```

Набор токенов одинаковый, но порядок разный.
Это и есть главная причина, по которой Transformer encoder не может обходиться без positional embedding.

Вход:
- padded последовательности целых token ids,

Выход:

- `y = 1`, если `7` стоит раньше `3`;

[cell 012 | markdown]
## Мини-теория

Минимальный encoder block:

$$
H_1 = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))
$$

$$
H_2 = \mathrm{LayerNorm}(H_1 + \mathrm{FFN}(H_1))
$$

Если positional embedding отсутствует, то attention знает “какие токены были”, но слабо знает “где они были”.
Для этой лабораторной это критично, потому что метка зависит именно от порядка.

def generate_order_dataset(num_samples, seq_len=SEQ_LEN, min_len=MIN_LEN, seed=SEED):
    """Генерирует синтетический набор для задачи чувствительности к порядку.

    Аргументы:
      num_samples: Число последовательностей для генерации.
      seq_len: Максимальная длина последовательности после дополнения.
      min_len: Минимальная полезная длина до дополнения.
      seed: Зерно случайности для воспроизводимости.

    Возвращает:
      Кортеж `(X, y, lengths)`, где:
        `X`: Матрица токенов формы `(num_samples, seq_len)`.
        `y`: Вектор бинарных меток формы `(num_samples,)`.
        `lengths`: Полезные длины до дополнения формы `(num_samples,)`.

    Исключения:
      ValueError: Если `min_len` или `seq_len` заданы некорректно.
    """
    if min_len < 2 or seq_len < min_len:
        raise ValueError('Ожидается 2 <= min_len <= seq_len.')

    rng = np.random.default_rng(seed)
    X = np.full((num_samples, seq_len), PAD_ID, dtype=np.int32)
    y = np.zeros((num_samples,), dtype=np.int32)
    lengths = np.zeros((num_samples,), dtype=np.int32)

    for i in range(num_samples):
        length = int(rng.integers(min_len, seq_len + 1))
        seq = rng.choice(filler_tokens, size=length, replace=True).astype(np.int32)
        pos_key, pos_value = rng.choice(length, size=2, replace=False)
        seq[pos_key] = KEY_TOKEN
        seq[pos_value] = VALUE_TOKEN
        X[i, :length] = seq
        y[i] = int(pos_key < pos_value)
        lengths[i] = length

    return X, y, lengths

class TokenAndPositionEmbedding(layers.Layer):
    """Складывает токенные и позиционные векторы."""

    def __init__(self, maxlen, vocab_size, embed_dim, **kwargs):
        """Инициализирует слой встраивания.

        Аргументы:
          maxlen: Максимальная длина последовательности.
          vocab_size: Размер словаря токенов.
          embed_dim: Размерность векторного представления.
          **kwargs: Дополнительные аргументы базового слоя Keras.

        Возвращает:
          Экземпляр слоя.

        Исключения:
          ValueError: Если `embed_dim` меньше 1.
        """
        super().__init__(**kwargs)
        if embed_dim < 1:
            raise ValueError('embed_dim должен быть положительным.')
        self.maxlen = maxlen
        self.vocab_size = vocab_size
        self.embed_dim = embed_dim
        self.token_emb = layers.Embedding(vocab_size, embed_dim, mask_zero=True)
        self.pos_emb = layers.Embedding(maxlen, embed_dim)
        self.supports_masking = True

    def call(self, inputs):
        """Возвращает сумму токенных и позиционных векторов.

        Аргументы:
          inputs: Целочисленные токены формы `(batch, time)`.

        Возвращает:
          Тензор формы `(batch, time, embed_dim)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        positions = tf.range(start=0, limit=tf.shape(inputs)[-1], delta=1)
        position_embeddings = self.pos_emb(positions)
        token_embeddings = self.token_emb(inputs)
        return token_embeddings + position_embeddings

    def compute_mask(self, inputs, mask=None):
        """Переадресует маску непустых токенов.

        Аргументы:
          inputs: Целочисленные токены формы `(batch, time)`.
          mask: Входная маска базового уровня.

        Возвращает:
          Булева маска формы `(batch, time)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        return self.token_emb.compute_mask(inputs)


def masked_average(x, mask):
    """Вычисляет среднее по времени с учётом маски.

    Аргументы:
      x: Тензор признаков формы `(batch, time, embed_dim)`.
      mask: Булева маска полезных позиций формы `(batch, time)`.

    Возвращает:
      Усреднённый тензор формы `(batch, embed_dim)`.

    Исключения:
      ValueError: Если входной тензор имеет неверный ранг.
    """
    if x.shape.rank != 3:
        raise ValueError('Ожидается ранг 3 для тензора признаков.')
    mask = tf.cast(mask, x.dtype)
    mask = tf.expand_dims(mask, axis=-1)
    summed = tf.reduce_sum(x * mask, axis=1)
    counts = tf.reduce_sum(mask, axis=1)
    return summed / tf.maximum(counts, 1.0)


class TransformerEncoderBlock(layers.Layer):
    """Минимальный блок кодировщика трансформера."""

    def __init__(self, embed_dim, num_heads, ff_dim, rate=0.1, **kwargs):
        """Инициализирует слои внимания и позиционно-независимой сети.

        Аргументы:
          embed_dim: Размерность входных признаков.
          num_heads: Число голов внимания.
          ff_dim: Размер скрытого слоя позиционно-независимой сети.
          rate: Доля выключаемых нейронов в прореживании.
          **kwargs: Дополнительные аргументы базового слоя Keras.

        Возвращает:
          Экземпляр блока кодировщика.

        Исключения:
          ValueError: Если `embed_dim` не делится на `num_heads`.
        """
        super().__init__(**kwargs)
        if embed_dim % num_heads != 0:
            raise ValueError('embed_dim должен делиться на num_heads без остатка.')
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.ff_dim = ff_dim
        self.rate = rate
        self.att = layers.MultiHeadAttention(
            num_heads=num_heads,
            key_dim=embed_dim // num_heads,
            dropout=rate,
        )
        self.ffn = keras.Sequential(
            [
                layers.Dense(ff_dim, activation='relu'),
                layers.Dense(embed_dim),
            ]
        )
        self.layernorm1 = layers.LayerNormalization(epsilon=1e-6)
        self.layernorm2 = layers.LayerNormalization(epsilon=1e-6)
        self.dropout1 = layers.Dropout(rate)
        self.dropout2 = layers.Dropout(rate)
        self.supports_masking = True

    def call(self, inputs, mask=None, training=None, return_attention_scores=False):
        """Выполняет прямой проход блока кодировщика.

        Аргументы:
          inputs: Входной тензор формы `(batch, time, embed_dim)`.
          mask: Булева маска формы `(batch, time)`.
          training: Признак режима обучения.
          return_attention_scores: Вернуть ли дополнительно веса внимания.

        Возвращает:
          Либо выходной тензор формы `(batch, time, embed_dim)`,
          либо кортеж `(output, attention_scores)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        attention_mask = None
        if mask is not None:
            key_mask = tf.cast(mask[:, tf.newaxis, :], dtype=tf.bool)
            query_mask = tf.cast(mask[:, :, tf.newaxis], dtype=tf.bool)
            attention_mask = query_mask & key_mask

        if return_attention_scores:
            attention_output, attention_scores = self.att(
                query=inputs,
                value=inputs,
                key=inputs,
                attention_mask=attention_mask,
                return_attention_scores=True,
                training=training,
            )
        else:
            attention_output = self.att(
                query=inputs,
                value=inputs,
                key=inputs,
                attention_mask=attention_mask,
                training=training,
            )

        attention_output = self.dropout1(attention_output, training=training)
        proj_input = self.layernorm1(inputs + attention_output)
        proj_output = self.ffn(proj_input)
        proj_output = self.dropout2(proj_output, training=training)
        output = self.layernorm2(proj_input + proj_output)

        if return_attention_scores:
            return output, attention_scores
        return output

    def compute_mask(self, inputs, mask=None):
        """Пробрасывает временную маску на следующий слой.

        Аргументы:
          inputs: Входной тензор признаков.
          mask: Входная булева маска.

        Возвращает:
          Та же маска без изменений.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        return mask

После этого блока модель должна:

- превращать их в embeddings с позициями;

    SEQ_LEN,
    VOCAB_SIZE,
    EMBED_DIM,
    name='token_and_position_embedding',

    NUM_HEADS,
    FF_DIM,
    rate=0.1,
    name='transformer_encoder_block',
)

x = embedding_layer(transformer_inputs)
x = encoder_layer(x, mask=padding_mask)
pooled = layers.Lambda(lambda args: masked_average(args[0], args[1]), name='masked_average')(

)

pooled = layers.Dense(32, activation='relu', name='hidden_head')(pooled)
outputs = layers.Dense(1, activation='sigmoid', name='label')(pooled)

model = keras.Model(transformer_inputs, outputs, name='transformer_order_classifier')

## Типичные ошибки (симптом -> причина -> исправление)

- `attention_scores` неожиданной формы -> перепутано ожидание выхода `MultiHeadAttention` -> помнить про `(batch, heads, T, T)`.

## Контракт данных

Вход:
- padded review-последовательности token ids,
- форма `X -> (N, T)`.

Выход:
- бинарная sentiment-метка `y -> (N,)`.

Здесь `PAD = 0`, поэтому padding mask можно строить как `tokens != 0`.

[cell 009 | markdown]
## Таблица форм тензоров

| Сущность | Смысл | Форма |
|---|---|---|
| `X_train` | padded review tokens | `(N, T)` |
| `padding_mask` | полезные позиции | `(N, T)` |
| `embeddings` | token + position embeddings | `(N, T, E)` |
| `attention_scores` | веса внимания | `(N, H, T, T)` |
| `pooled` | один вектор на review | `(N, E)` |
| `y_pred` | вероятность positive class | `(N, 1)` |

[cell 010 | markdown]
## Шпаргалка по обозначениям и формам

- `N` — число review.
- `T` — длина review после padding.
- `E` — размер embedding / model dimension.
- `H` — число attention heads.
- `V` — размер словаря.

Для этой лабораторной важны две привычки:
- проверять mask;
- не пытаться интерпретировать attention на всём review целиком, если оно слишком длинное.

## Контракт модели

## Мини-теория

Encoder-only Transformer для классификации можно читать так:
- токены получают embeddings и позиции;
- encoder block смешивает информацию между позициями;
- затем всё сворачивается в один вектор на объект;
- classifier head выдаёт sentiment-вероятность.

На практике это уже рабочий пример того, как self-attention заменяет рекуррентный проход в `many-to-one` задаче.

## Ручной разбор одного примера

Review после tokenization — это уже не слова, а целочисленные ids.

С точки зрения формы всё выглядит знакомо:

```text
review tokens -> (T,)
mask          -> (T,)
embeddings    -> (T, E)
```

Новое здесь не shape, а то, что attention должен работать на реальном тексте, а не на короткой synthetic последовательности.

class TokenAndPositionEmbedding(layers.Layer):
    """Складывает токенные и позиционные векторы."""

    def __init__(self, maxlen, vocab_size, embed_dim, **kwargs):
        """Инициализирует слой встраивания.

        Аргументы:
          maxlen: Максимальная длина последовательности.
          vocab_size: Размер словаря токенов.
          embed_dim: Размерность векторного представления.
          **kwargs: Дополнительные аргументы базового слоя Keras.

        Возвращает:
          Экземпляр слоя.

        Исключения:
          ValueError: Если `embed_dim` меньше 1.
        """
        super().__init__(**kwargs)
        if embed_dim < 1:
            raise ValueError('embed_dim должен быть положительным.')
        self.maxlen = maxlen
        self.vocab_size = vocab_size
        self.embed_dim = embed_dim
        self.token_emb = layers.Embedding(vocab_size, embed_dim, mask_zero=True)
        self.pos_emb = layers.Embedding(maxlen, embed_dim)
        self.supports_masking = True

    def call(self, inputs):
        """Возвращает сумму токенных и позиционных векторов.

        Аргументы:
          inputs: Целочисленные токены формы `(batch, time)`.

        Возвращает:
          Тензор формы `(batch, time, embed_dim)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        positions = tf.range(start=0, limit=tf.shape(inputs)[-1], delta=1)
        position_embeddings = self.pos_emb(positions)
        token_embeddings = self.token_emb(inputs)
        return token_embeddings + position_embeddings

    def compute_mask(self, inputs, mask=None):
        """Переадресует маску непустых токенов.

        Аргументы:
          inputs: Целочисленные токены формы `(batch, time)`.
          mask: Входная маска базового уровня.

        Возвращает:
          Булева маска формы `(batch, time)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        return self.token_emb.compute_mask(inputs)


def masked_average(x, mask):
    """Вычисляет среднее по времени с учётом маски.

    Аргументы:
      x: Тензор признаков формы `(batch, time, embed_dim)`.
      mask: Булева маска полезных позиций формы `(batch, time)`.

    Возвращает:
      Усреднённый тензор формы `(batch, embed_dim)`.

    Исключения:
      ValueError: Если входной тензор имеет неверный ранг.
    """
    if x.shape.rank != 3:
        raise ValueError('Ожидается ранг 3 для тензора признаков.')
    mask = tf.cast(mask, x.dtype)
    mask = tf.expand_dims(mask, axis=-1)
    summed = tf.reduce_sum(x * mask, axis=1)
    counts = tf.reduce_sum(mask, axis=1)
    return summed / tf.maximum(counts, 1.0)


class TransformerEncoderBlock(layers.Layer):
    """Минимальный блок кодировщика трансформера."""

    def __init__(self, embed_dim, num_heads, ff_dim, rate=0.1, **kwargs):
        """Инициализирует слои внимания и позиционно-независимой сети.

        Аргументы:
          embed_dim: Размерность входных признаков.
          num_heads: Число голов внимания.
          ff_dim: Размер скрытого слоя позиционно-независимой сети.
          rate: Доля выключаемых нейронов в прореживании.
          **kwargs: Дополнительные аргументы базового слоя Keras.

        Возвращает:
          Экземпляр блока кодировщика.

        Исключения:
          ValueError: Если `embed_dim` не делится на `num_heads`.
        """
        super().__init__(**kwargs)
        if embed_dim % num_heads != 0:
            raise ValueError('embed_dim должен делиться на num_heads без остатка.')
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.ff_dim = ff_dim
        self.rate = rate
        self.att = layers.MultiHeadAttention(
            num_heads=num_heads,
            key_dim=embed_dim // num_heads,
            dropout=rate,
        )
        self.ffn = keras.Sequential(
            [
                layers.Dense(ff_dim, activation='relu'),
                layers.Dense(embed_dim),
            ]
        )
        self.layernorm1 = layers.LayerNormalization(epsilon=1e-6)
        self.layernorm2 = layers.LayerNormalization(epsilon=1e-6)
        self.dropout1 = layers.Dropout(rate)
        self.dropout2 = layers.Dropout(rate)
        self.supports_masking = True

    def call(self, inputs, mask=None, training=None, return_attention_scores=False):
        """Выполняет прямой проход блока кодировщика.

        Аргументы:
          inputs: Входной тензор формы `(batch, time, embed_dim)`.
          mask: Булева маска формы `(batch, time)`.
          training: Признак режима обучения.
          return_attention_scores: Вернуть ли дополнительно веса внимания.

        Возвращает:
          Либо выходной тензор формы `(batch, time, embed_dim)`,
          либо кортеж `(output, attention_scores)`.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        attention_mask = None
        if mask is not None:
            key_mask = tf.cast(mask[:, tf.newaxis, :], dtype=tf.bool)
            query_mask = tf.cast(mask[:, :, tf.newaxis], dtype=tf.bool)
            attention_mask = query_mask & key_mask

        if return_attention_scores:
            attention_output, attention_scores = self.att(
                query=inputs,
                value=inputs,
                key=inputs,
                attention_mask=attention_mask,
                return_attention_scores=True,
                training=training,
            )
        else:
            attention_output = self.att(
                query=inputs,
                value=inputs,
                key=inputs,
                attention_mask=attention_mask,
                training=training,
            )

        attention_output = self.dropout1(attention_output, training=training)
        proj_input = self.layernorm1(inputs + attention_output)
        proj_output = self.ffn(proj_input)
        proj_output = self.dropout2(proj_output, training=training)
        output = self.layernorm2(proj_input + proj_output)

        if return_attention_scores:
            return output, attention_scores
        return output

    def compute_mask(self, inputs, mask=None):
        """Пробрасывает временную маску на следующий слой.

        Аргументы:
          inputs: Входной тензор признаков.
          mask: Входная булева маска.

        Возвращает:
          Та же маска без изменений.

        Исключения:
          RuntimeError: Не выбрасывается в штатном режиме.
        """
        return mask

# Трансформер-кодировщик (Transformer encoder): следующий шаг после внимания

Место в линии курса:
- предыдущий шаг: `02-Attention / ЛР01`;
- текущие лабораторные этой темы: `03-Transformer / ЛР01`, `03-Transformer / ЛР02`;
- общий порядок курса: Шаг 5 и Шаг 6.

## 1. Что меняется после перекрёстного внимания

В `02-Attention / ЛР01` декодер использовал память кодировщика:

$$
\text{context}_t = \mathrm{Attention}(q_t^{dec}, K^{enc}, V^{enc})
$$

Это перекрёстное внимание (cross-attention):
- запросы приходят из одной последовательности;
- ключи и значения приходят из другой.

Дальше курс делает следующий шаг: разрешает последовательности смотреть на саму себя.

Так появляется самовнимание (self-attention):
- каждый токен строит собственные `query`, `key`, `value`;
- токены оценивают полезность друг друга;
- дальние зависимости связываются напрямую, без рекуррентного прохода.

## 2. Масштабированное скалярное внимание

Пусть матрица признаков последовательности равна:

$$
X \in \mathbb{R}^{T \times d_{model}}
$$

Тогда:

$$
Q = XW_Q, \quad K = XW_K, \quad V = XW_V
$$

Оценки совместимости вычисляются так:

$$
S = \frac{QK^\top}{\sqrt{d_k}}
$$

Потом по каждой строке применяется `softmax`:

$$
A = \mathrm{softmax}(S)
$$

И выход блока внимания:

$$
\mathrm{Attention}(Q, K, V) = AV
$$

### Почему делим на $\sqrt{d_k}$

Если компоненты `q` и `k` независимы, центрированы и имеют дисперсию порядка 1, то

$$
q \cdot k = \sum_{i=1}^{d_k} q_i k_i
$$

имеет дисперсию порядка $d_k$. Значит типичный масштаб скалярного произведения растёт как $\sqrt{d_k}$.

Если не делить на $\sqrt{d_k}$, при большом `d_k` значения перед `softmax` становятся слишком крупными по модулю, распределение переходит в почти вырожденный режим, а градиенты быстро затухают.

Деление на $\sqrt{d_k}$ возвращает оценки к стабильной шкале и делает обучение устойчивее.

Бытовая аналогия: если в одном отчёте суммы указаны в рублях, а в другом в копейках, сравнение «как есть» вводит в заблуждение. Масштабирование — это приведение к одной шкале перед сравнением.

## 3. Почему трансформер не может жить без позиции

Самовнимание само по себе не хранит порядок токенов. Если переставить токены местами, модель видит тот же набор векторов, но не знает, кто стоял раньше.

Поэтому к токенным представлениям добавляют позиционную информацию:
- синусоидальное позиционное кодирование (sinusoidal positional encoding);
- обучаемые позиционные векторы (learned positional embedding).

В этом блоке курса используется второй вариант: он проще для первой практики и хорошо поддерживается в Keras.

## 4. Многоголовое внимание

Одна голова внимания — один способ выделять связи между позициями.

При нескольких головах:

$$
\mathrm{head}_i = \mathrm{Attention}(Q_i, K_i, V_i)
$$

$$
\mathrm{MHA}(X) = \mathrm{Concat}(\mathrm{head}_1, \dots, \mathrm{head}_h)W_O
$$

Интуиция:
- одна голова может подчёркивать локальные связи;
- другая — дальние зависимости;
- третья — границы смысловых фрагментов.

## 5. Блок кодировщика трансформера

Минимальный блок состоит из двух подслоёв:
1. многоголовое самовнимание;
2. позиционно-независимая двухслойная сеть.

Оба подслоя окружены остаточными связями и нормализацией:

$$
H_1 = \mathrm{LayerNorm}(X + \mathrm{MHA}(X))
$$

$$
H_2 = \mathrm{LayerNorm}(H_1 + \mathrm{FFN}(H_1))
$$

где

$$
\mathrm{FFN}(h) = W_2 \sigma(W_1 h + b_1) + b_2
$$

Практический смысл:
- внимание смешивает информацию между позициями;
- позиционно-независимая сеть дообрабатывает каждый временной шаг;
- остаточная связь помогает сохранять сигнал;
- нормализация стабилизирует распределение активаций.

## 6. Маска дополнения и формы тензоров

Если последовательности дополнены `PAD`-токенами, внимание не должно считать их содержательными.

Иначе:
- часть веса уходит в пустые позиции;
- карты внимания становятся шумными;
- итоговое усреднение искажается.

### Формализация маски

Пусть базовая маска полезных токенов:

$$
m \in \{0,1\}^{B \times T}, \quad m_{b,t}=1 \Leftrightarrow \text{позиция непустая}
$$

Для внимания нужно запретить участие пустых позиций и как ключей, и как запросов. Поэтому формируется маска

$$
M_{b,i,j} = m_{b,i} \land m_{b,j}
$$

с формой `(B, T, T)`. На практике её удобно собирать как логическое «И» двух тензоров:
- `query_mask = m[:, :, None]`;
- `key_mask = m[:, None, :]`;
- `attention_mask = query_mask & key_mask`.

Это устраняет ложные связи с `PAD` и делает интерпретацию карт внимания корректной.

## 7. Почему в этой теме только кодировщик

Полный трансформер обычно включает:
- кодировщик;
- декодер;
- маскированное самовнимание в декодере;
- перекрёстное внимание декодера к кодировщику.

Для первого знакомства это слишком плотный шаг. Поэтому здесь фиксируется `encoder-only`-версия:
- проще переход от внимания к самовниманию;
- проще увидеть роль позиции;
- проще удержать вычислительную нагрузку на центральном процессоре.

## 8. Как это проявляется в лабораторных

### `03-Transformer / ЛР01`

Синтетическая задача, чувствительная к порядку:
- один и тот же набор токенов может давать разные метки;
- следовательно, модели нужно знать не только «что пришло», но и «в каком порядке».

### `03-Transformer / ЛР02`

Классификация `IMDB`:
- та же архитектурная идея переносится на реальные тексты;
- длина входа больше;
- метка одна на последовательность;
- маска дополнения остаётся обязательной и во внимании, и в пуллинге.

## 9. Что должно остаться после темы

- самовнимание — механизм взаимодействия токенов внутри одной последовательности;
- позиционная информация нужна, чтобы модель не теряла порядок;
- масштабирование `1/\sqrt{d_k}` стабилизирует обучение через нормировку шкалы оценок;
- корректная маска должна учитывать и запросы, и ключи;
- следующий логический шаг — блок [../../04-Autoregression/lab/README.md](../../04-Autoregression/lab/README.md) с декодерным трансформером и причинной маской.

## 11. Расшифровка каждой закорючки: от cross-attention к self-attention

Этот раздел не заменяет формулы выше. Он добавляет медленное чтение каждой части transformer encoder: что значит символ, какой tensor должен получиться и где студент увидит это в лабораторной.

### 11.1 Что это значит словами

В `02-Attention` decoder спрашивал encoder: “на какую входную позицию смотреть?”.

В transformer encoder каждый токен спрашивает другие токены той же последовательности: “кто из вас мне полезен, чтобы обновить моё представление?”.

Если совсем с нуля:
- `self` означает “внутри той же последовательности”;
- `attention` означает “таблица весов между позициями”;
- `positional` нужно потому, что attention без позиции не знает порядок.

Если профессионально:
- self-attention заменяет рекуррентную передачу состояния прямыми попарными взаимодействиями;
- positional embedding добавляет информацию, нарушающую перестановочную симметрию набора токенов.

### 11.2 Что означает каждый символ

| Символ | Смысл | Shape-contract |
|---|---|---|
| $X$ | матрица представлений токенов | `(T, d_model)` или `(batch, T, d_model)` |
| $W_Q,W_K,W_V$ | обучаемые проекции | переводят `d_model` в `d_k`/`d_v` |
| $Q$ | запросы всех позиций | `(T, d_k)` |
| $K$ | ключи всех позиций | `(T, d_k)` |
| $V$ | значения всех позиций | `(T, d_v)` |
| $S$ | attention logits до softmax | `(T, T)` |
| $A$ | attention weights после softmax | `(T, T)` |
| $M$ | mask | broadcast к attention logits |

### 11.3 Разбор строки формулы

$$
S = \frac{QK^\top}{\sqrt{d_k}}
$$

Чтение по шагам:
1. $QK^\top$ сравнивает каждый запрос с каждым ключом.
2. Результат имеет форму `(T, T)`: строка — кто спрашивает, столбец — на кого смотрят.
3. Деление на $\sqrt{d_k}$ стабилизирует масштаб перед softmax.

Дальше:

$$
A = \mathrm{softmax}(S)
$$

Softmax применяется по key-оси: в каждой строке веса суммируются в `1`.

### 11.4 Shape-contract для лабораторных

```text
tokens           : (batch, T)
token_embeddings : (batch, T, d_model)
position_vectors : (T, d_model) или (batch, T, d_model)
padding_mask     : (batch, T)
attention_scores : (batch, heads, T, T)
pooled_output    : (batch, d_model)
classifier_out   : (batch, 1)
```

### 11.5 Мини-числовой пример маски

Если:

```text
padding_mask = [1, 1, 0]
```

то позиция `2` является `PAD`. Attention не должен считать её содержательной ни как key, ни как query в диагностике.

Идея формулы:

$$
M_{i,j}=m_i \land m_j
$$

Расшифровка:
- $m_i$ проверяет, является ли строка/query валидной;
- $m_j$ проверяет, является ли столбец/key валидным;
- $\land$ означает “оба условия должны быть истинны”.
